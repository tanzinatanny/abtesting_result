Goal

We have made A and B version of the interface of a cactus store and ran user testing on them to measure statistical significance.

Hypothesis

Time to completion


Null Hypothesis : Any changes between the A and B version of the website does not change the time to complete for the task of adding items in the cart

Alternative Hypothesis : Version A works better by reducing time to completion of the task of adding items in the the cart within a constraint than version B

Reason: Version A has the items in a horizontal list view and the cart is moved to the top within the same page view of the website whereas in the version B, each item takes up the whole page view and uses scroll view until it reaches the cart is at the bottom of the webpage. Therefore, the task of adding items in the cart should take more time in version B, thus version A should work better in this case.


Return rate


Null Hypothesis : Any changes between the A and B version of the website does not change the return rate for the task of adding items in the cart

Alternative Hypothesis : Version A works better by reducing time to completion of the task of adding items in the the cart within a constraint than version B

Reason: Vesrion A has the cart icon moved to the top within the same page view of the website along with the listed items whereas in the version B, the cart is at the bottom of the webpage. Therefore, version A facilitates memorability as the user can see the number of items added in the cart easily unlike in version B.

Data Collection Process

Collected the raw data from the student activity during the studio-6(20 users) and processed the data by cleaning the raw data into usable data. For this, I sorted the data sheet by users and calculated the time spent by each user on the website by subtracting the entry or page load time from the exit time. The return rate was calculated by looking into the users activity of making a return from the cart and calculating the occurances as caterorical data item.
